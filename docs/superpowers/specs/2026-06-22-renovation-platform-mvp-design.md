# 装修平台 MVP 设计文档

## 产品定位

本产品是一款先面向 iOS 和 App Store 发布的装修服务平台，核心定位是“业主直连工长”。

首版不是开放式需求发布平台。业主不需要发布一条装修需求后等待工人抢单或报价，而是通过标准化的项目类别和工长展示入口了解服务，再预约工长。项目类别包括全屋翻新、旧房翻新、厨房改造、卫生间改造等。

首版重点跑通：业主浏览服务、预约工长、平台客服确认、后台派单、工长接单、在线沟通、施工群聊、阶段照片确认、每日水印照片留痕、师傅动态审核发布。

首版不做在线支付、定金、分阶段付款、电子合同、完整报价系统和材料采购系统。这些能力放到后续版本。

## 产品端结构

### 业主端 iOS App

业主端支持匿名浏览，只有在预约工长时才需要手机号验证码登录。

首版能力：

- 浏览已开通城市和项目类别。
- 查看项目介绍页。
- 查看工长基础信息、案例和师傅动态。
- 从项目介绍页预约工长。
- 与工长在线沟通，支持文字、图片和语音识别文字。
- 确认合作后进入施工项目群聊。
- 查看施工阶段。
- 确认或驳回工长提交的阶段照片。
- 查看每日完工水印照片。
- 浏览公开师傅动态，点赞和评价。
- 联系平台客服。

### 工长端 iOS App

工长端是独立 App。工长使用手机号验证码登录，必须通过后台审核后才能接收平台派单。

首版能力：

- 手机号验证码登录。
- 提交和维护工长资料。
- 维护个人主页。
- 上传施工案例。
- 接收平台派单。
- 接受或拒绝派单。
- 与业主在线沟通，支持文字、图片和语音识别文字。
- 参与施工项目群聊。
- 按阶段上传施工照片。
- 每天完工后使用水印相机上传照片。
- 将符合规则的基础施工内容提交为公开师傅动态。

### Web 管理后台

后台供平台客服、监理和运营人员使用，是首版业务闭环的核心。

首版能力：

- 管理开通城市和服务区域。
- 管理项目类别。
- 管理项目介绍页内容。
- 按项目类别配置施工阶段模板。
- 审核工长注册资料、主页内容和案例。
- 处理业主预约线索。
- 客服确认业主需求。
- 将预约派单给合适工长。
- 查看业主与工长的聊天记录。
- 查看施工项目群聊。
- 查看阶段照片和每日水印照片。
- 审核师傅动态后决定是否公开。
- 管理点赞、评价和举报内容。

## 业主端核心流程

1. 业主打开 App，可先匿名浏览。
2. 首页展示两类入口：
   - 项目类别：全屋翻新、旧房翻新、厨房改造、卫生间改造等。
   - 工长发现：展示已审核工长、案例和师傅动态。
3. 业主点击项目类别后，进入项目介绍页。
4. 项目介绍页展示：
   - 首屏服务卖点。
   - 透明收费、透明施工、透明用料等信任点。
   - 服务流程。
   - 服务内容。
   - 工价或工艺说明。
   - 材料说明。
   - 业主评价。
   - 工长或施工案例。
   - 底部固定“平台客服”和“预约工长”操作。
5. 业主点击“预约工长”。
6. 如果未登录，先使用手机号验证码登录。
7. 业主提交预约表单。
8. 后台生成预约线索。
9. 平台客服联系业主确认需求。
10. 后台将预约派单给合适工长。
11. 工长接单后，业主和工长可在 App 内沟通。
12. 业主确认合作后，预约转为施工项目。
13. 系统自动生成施工项目群聊。
14. 工长按阶段上传照片，并每天上传水印完工照片。
15. 业主确认或驳回阶段提交。
16. 符合规则的施工内容经后台审核后，可公开为师傅动态。

## 预约表单

首版预约表单收集客服和派单所需的基础信息，但不要求业主填写预算范围。

字段：

- 城市/服务区域。
- 小区或详细地址。
- 项目类别。
- 房屋面积。
- 户型：几室几厅几厨几卫。
- 房屋状态：新房或旧房。
- 改造范围：局改或全改。
- 期望上门时间。
- 手机号。
- 备注。

预算范围不放进首版预约表单。价格由客服和工长在后续沟通中确认。

## 隐私规则

平台需要保护业主电话和地址隐私。

电话规则：

- 业主真实手机号由平台保存。
- 工长不能直接看到业主真实手机号。
- 工长通过虚拟号码联系业主。

地址规则：

- 完整地址由平台后台保存。
- 派单确认前，工长只能看到脱敏地址，例如 `XX市 XX区 XX小区`。
- 预约确认并派单成功后，仅接单工长可以看到完整地址。
- 公开师傅动态不能展示完整地址、门牌号、业主手机号或业主身份信息。

## 沟通体系

业主端和工长端都支持文字、图片和语音识别文字。

沟通分为两层：

- 预约沟通：围绕预约和合作确认进行的一对一沟通。
- 施工项目群聊：确认合作后自动创建，用于施工过程协同。

施工群聊成员：

- 业主。
- 工长/工人。
- 平台监理。
- 平台客服。

后台可以查看业主与工长聊天记录，也可以查看施工项目群聊，用于服务质量管理、纠纷处理和运营监督。

## 施工阶段管理

施工阶段按项目类别配置模板。

示例：

- 全屋翻新：开工、拆改、水电、防水、泥瓦、木工、油漆、安装、竣工验收。
- 厨房改造：拆除、水电调整、防水、贴砖、吊顶/橱柜衔接、安装、验收。
- 卫生间改造：拆除、防水、水电调整、贴砖、洁具安装、验收。

后台为每个项目类别配置阶段模板。业主确认合作后，系统根据项目类别自动生成施工阶段。

每个阶段包含：

- 阶段名称。
- 工长上传的照片。
- 业主确认状态。
- 驳回原因或沟通记录。
- 确认或驳回时间。

业主可以确认阶段，也可以驳回并说明原因。阶段记录保存在项目历史和后台中。

## 每日水印完工照片

项目开始后，工长每天完工需要使用水印相机上传照片。

水印信息包括：

- 拍摄时间。
- 项目城市/小区或脱敏地址。
- 工长姓名。
- 项目或阶段名称。
- 工作内容标签。

每日水印照片对业主、项目群聊成员和后台可见。符合公开规则的照片可以提交为师傅动态，等待后台审核。

## 师傅动态

产品包含公开师傅动态，形式类似施工版朋友圈。它不是普通生活社交动态，而是用于展示施工过程、工艺能力和工长可信度。

允许展示的内容：

- 水电改造过程。
- 防水施工过程。
- 贴砖过程。
- 木工吊顶。
- 墙面处理和油漆。
- 安装收口。
- 其他经平台认可的基础施工过程。

发布流程：

1. 工长上传每日水印照片。
2. 工长或平台将符合条件的内容标记为可公开动态。
3. 后台审核内容。
4. 审核人员确认内容为基础施工内容，并且没有暴露业主隐私。
5. 平台发布动态。
6. 其他业主和用户可以浏览、点赞、评价。

禁止公开的内容：

- 完整地址或门牌号。
- 业主姓名、手机号或可识别隐私信息。
- 私聊截图或聊天记录。
- 与施工无关的业主私人生活物品。
- 非施工相关内容。
- 后台审核不通过的内容。

## 工长信任资产

工长资料和动态用于建立业主信任。

工长信任资产包括：

- 头像和展示名称。
- 身份认证状态。
- 主要服务项目。
- 从业年限。
- 服务城市/区域。
- 施工案例。
- 公开师傅动态。
- 业主评价。
- 平台审核状态。

工长主页、案例和公开展示内容都需要经过平台审核。

## 后台运营流程

后台是首版强运营模式的核心。

关键流程：

- 城市开通：配置可预约城市和服务区域。
- 内容管理：配置项目类别和项目介绍页。
- 工长审核：审核身份、服务项目、案例和个人主页。
- 预约处理：查看新预约、联系客服确认、更新状态。
- 派单管理：将预约派给合适工长。
- 隐私控制：管理地址展示规则和虚拟号码联系规则。
- 聊天监管：查看预约聊天和施工群聊。
- 阶段监管：查看阶段照片和业主确认状态。
- 动态审核：审核师傅动态后公开或驳回。
- 评价管理：管理点赞、评价和举报内容。

## 账号与认证

业主端：

- 允许匿名浏览。
- 预约工长时需要手机号验证码登录。

工长端：

- 必须手机号验证码登录。
- 必须通过平台审核后才能接收派单。

首版不做 Apple 登录、微信登录、复杂账号绑定和支付账号体系。

## 发布目标

首版先做 iOS，并以 App Store 发布为目标。

开发方式：

- 业主端和工长端使用原生 iOS 开发，推荐 Swift/SwiftUI。
- 业主端和工长端是两个独立 App。
- 后台为 Web 管理后台。

App Store 上架前需要准备：

- 隐私政策。
- 用户协议。
- 客服联系方式。
- 用户生成内容的审核说明。
- 师傅动态内容审核机制。
- 账号注销入口。
- 相机、相册、麦克风/语音识别、通知、位置等权限说明。

## 首版不包含

以下能力不进入 MVP：

- 在线支付。
- 预约金或定金。
- 分阶段付款。
- 电子合同签署。
- 完整报价系统。
- 材料采购和下单流程。
- 自动匹配工长。
- 全国开放预约。
- 普通用户自定义每个项目的施工阶段。
- Android App。

这些能力可以在预约、派单、沟通、施工留痕和师傅动态跑通后逐步加入。

## 核心闭环

首版要完整跑通这条链路：

业主浏览服务 -> 预约工长 -> 平台客服确认 -> 后台派单 -> 工长接单 -> 业主和工长沟通 -> 确认合作 -> 自动生成施工群聊 -> 工长上传阶段照片和每日水印照片 -> 业主确认施工进度 -> 后台审核公开施工动态 -> 师傅动态沉淀为工长信任资产。
