# 装修报价网页设计规格

## 目标

建设一个可公网部署的独立装修报价网页，供团队成员使用统一团队密码登录，共同维护装修报价单。系统以空间为报价大项，以施工项目为小项，支持从项目库选择项目、编辑工程量与单价、实时汇总金额、保存报价并打印为 PDF。

首版先完成可运行网站。公网部署留到网站完成后进行，届时由用户注册免费的 Vercel 与 Supabase 账号并配置环境变量。

## 首版范围

首版包含：

- 团队密码登录。
- 报价记录列表与搜索。
- 新建、编辑、复制和删除报价。
- 按空间组织报价项目。
- 从项目库批量添加小项。
- 项目库的新增、编辑和停用。
- 综合价与人工费、材料费拆分价两种计价方式。
- 管理费、设计费、优惠等报价调整项。
- 自动保存与保存状态提示。
- 正式报价预览和浏览器打印/PDF。
- 适配电脑和手机。

首版不包含：

- 客户公开分享链接。
- 团队成员个人账号和权限分级。
- 报价审批流程。
- 完整版本历史和回滚。
- 在线签约、支付或开票。

## 信息架构

### 登录页

团队成员输入统一密码。服务端验证成功后写入安全会话 Cookie。未登录访问业务页面时返回登录页。

### 报价记录页

展示报价编号、客户、小区、面积、报价总额、状态和更新时间。支持关键词搜索、新建报价、复制报价、打开编辑和删除。删除前必须二次确认。

### 报价编辑页

桌面端采用三栏混合工作台：

- 左侧：空间导航、每个空间的项目数量、添加空间。
- 中间：当前空间信息与项目明细表。
- 右侧：人工费、材料费、其他费用、空间占比和报价总额。

手机端改为单列卡片布局：顶部横向空间选择，中间项目卡片，底部或折叠区域显示汇总。

### 项目库

从编辑页打开项目库时，默认按当前空间筛选。用户可搜索、勾选多个项目并批量加入当前空间。

项目库管理页支持新增、修改和停用项目。项目包含适用空间、名称、施工说明、单位、默认计价方式、默认综合单价、默认人工单价、默认材料单价和启用状态。

### 正式报价预览

打印版包含公司/品牌信息、报价编号、客户与项目基础信息、按空间分组的项目明细、费用汇总、备注和日期。打印时隐藏编辑控件，并通过浏览器打印功能输出 PDF。

## 报价数据与计算

### 报价基础信息

- 报价编号。
- 客户姓名与电话。
- 小区或项目地址。
- 建筑面积。
- 装修类型。
- 报价状态：草稿或已确认。
- 报价备注。
- 创建与更新时间。
- 并发更新版本号。

### 空间

默认可使用客厅/餐厅、主卧、次卧、厨房、卫生间、阳台和全屋公共项。用户可以添加、重命名、排序和删除空间。每个空间可保存地面面积、墙面面积及备注。

### 报价项目

每个报价项目保存项目库来源及报价快照：名称、施工说明、单位、工程量、计价方式、综合单价、人工单价、材料单价、备注和排序。

保存快照意味着后续修改项目库价格不会改变历史报价。用户仍可在单张报价内手动修改价格。

### 计价公式

- 综合价模式：项目金额 = 工程量 × 综合单价。
- 拆分价模式：项目金额 = 工程量 ×（人工单价 + 材料单价）。
- 空间小计 = 该空间所有项目金额之和。
- 报价小计 = 所有空间小计之和。
- 最终总价 = 报价小计 + 正向调整项 - 优惠项。

金额统一保存为“分”的整数，界面显示为人民币元，避免浮点误差。工程量允许最多两位小数。

## 关键交互

1. 用户新建报价并填写客户、小区、面积和装修类型。
2. 系统生成报价编号及默认空间。
3. 用户选择空间，打开项目库并批量添加所需小项。
4. 用户填写工程量，选择综合价或拆分价，并可修改预设价格。
5. 页面实时计算项目金额、空间小计和最终总价。
6. 编辑内容短暂延迟后自动保存，页面显示“保存中”“已保存”或“保存失败”。
7. 用户可进入正式报价预览，使用浏览器打印或另存为 PDF。

## 数据架构

使用 Next.js 构建响应式前端和服务端接口，使用 Supabase PostgreSQL 保存共享数据，使用 Vercel 部署网站。

核心数据表：

- `quotes`：报价基础信息、汇总字段、状态和版本号。
- `quote_spaces`：报价空间及排序。
- `quote_items`：报价项目快照及计价字段。
- `quote_adjustments`：管理费、设计费、优惠等调整项。
- `catalog_items`：项目库。

报价与空间、空间与项目均为一对多关系。所有写入通过受保护的服务端接口完成，不在浏览器暴露数据库管理密钥。

## 安全与会话

- 团队密码以服务端环境变量提供，不提交到代码库。
- 服务端使用安全哈希比对密码。
- 登录成功后签发带有效期、`HttpOnly`、`Secure` 和 `SameSite=Lax` 属性的会话 Cookie。
- 业务页面和写接口均校验会话。
- 数据库凭据仅存在于服务端环境变量。
- 所有输入在服务端进行长度、类型和数值范围验证。

统一团队密码满足首版需求，但不能识别具体操作者。个人账号与操作审计属于后续升级。

## 并发与错误处理

- 报价保存采用版本号进行乐观并发控制。
- 若他人已更新当前报价，服务端拒绝旧版本覆盖，页面提示“报价已被他人更新”，用户刷新后继续。
- 自动保存失败时保留当前浏览器内编辑状态，显示重试按钮，不清空内容。
- 网络恢复后允许用户再次保存。
- 删除报价或空间必须二次确认。
- 项目库项目被停用后，不影响已加入历史报价的项目快照。
- 无效金额、负数工程量和缺失必填信息在提交前提示并阻止保存。

## 视觉设计

整体采用沉稳、可信的装修行业视觉：深墨绿色作为主色，暖金色作为强调色，米白背景和白色卡片承载信息。避免过度装饰，优先保证大量报价数据的可读性。

桌面端保持空间导航、明细编辑和汇总金额同屏；移动端以触控友好的卡片和横向空间导航为主。项目名称与工程量优先展示，详细说明和拆分价格可展开查看。

布局预览文件为项目根目录的 `quote-layout-mockup.html`。

## 测试与验收

### 自动测试

- 综合价与拆分价计算。
- 调整项与最终总价计算。
- 金额精度和工程量小数处理。
- 团队密码登录和未登录拦截。
- 报价、空间和项目的新增、修改、删除。
- 自动保存成功、失败重试和并发冲突。
- 项目库停用不改变历史报价。

### 浏览器验收

- 桌面端三栏布局在常见宽度下无横向溢出。
- 手机端可以完成新建报价、添加项目、改价和查看总价。
- 项目库可搜索并批量选择。
- 报价记录可搜索、复制和删除。
- 正式报价打印分页合理，编辑控件不会出现在 PDF 中。
- 刷新页面后数据仍存在，另一浏览器登录后能看到同一批报价。

## 部署前置条件

实现和本地验证完成后，用户需要：

1. 注册 Vercel 账号。
2. 注册 Supabase 账号并创建项目。
3. 提供或在部署界面配置数据库地址、服务端密钥、团队密码哈希和会话密钥。
4. 执行项目内提供的数据库建表与种子数据脚本。
5. 部署后获得可发送给同事的 HTTPS 公网网址。

账号注册与外部服务配置由用户授权后进行；在此之前，本地开发使用可替换的数据适配层和测试数据库完成。
